4. Competitors

This project introduced an interesting situation. Before starting it, I looked in the Internet for already existing free software libraries that provided the functional requirements described in this document. Unfortunately, I wasn't able to find any, but obviously I was wrong since there was at least one: Jakarta Commons' Betwixt.

4.1. Jakarta Commons Betwixt

I wasn't aware of the fact that Betwixt existed until a friend of mine noticed if Betwixt and gave me that information. That happened just after most of the desired features were accomplished.

Ironically, the bad news are not really that bad. It's just knowing that I should have taken a deeper look at the existing alternatives. But afterwards, the "discover" was interesting, since I could evaluate if Undigester introduced any advantage over Betwixt, and if not no more effort would be dedicated to improving it.

The decision is not fully taken, tough. It's clear that Betwixt is a more mature module, and would have been used from the beginning, but I'm not sure it follows the same design as Digester. Although it provides direct compatibility with Digester rules, what is very useful, we'll keep Undigester as XML serialization API because it follows all ACM-SL standard and allows reuse of Digester's know-how.

In fact, the main application of Undigester, WebExtension, showed that Undigester wasn't usable to serialize web.xml information. The problem was related to the serialization algorithm itself, and its solution was not easy. Under such circumstances, WebExtension begun testing how to use Betwixt. After some attempts, the results were not satisfactory for WebExtension requirements. Without detailing the issues too much, it seemed that Betwixt doesn't provide an easy way of including PCDATA information provided by a class, without explicit element for such class. Since some of the classes used in WebExtension are reused and expected to be serialized in different ways depending on the information itself (not just the class), I found no way to make it work. Additionally, at the time of this writing, after searching at Betwixt mailing lists, I haven't been able to figure out how to omit an element representing an array or collection, only when such collection is empty.

So I decided again to focus the effort on debugging the recursive algorithm to find out how to solve Undigester bug.
